# 세션별 상세 학습자료 집필·검증 워크플로우

이 문서는 기존의 개요형 수업을 초급부터 전문가까지 이어지는 원자적 학습 세션으로 다시 만드는 공통 계약이다. 각 세션은 다른 세션의 설명을 생략의 근거로 삼지 않는다. 필요한 앞 개념은 짧게 다시 설명하고, 더 깊은 설명은 명시적인 링크로 연결한다.

## 1. 세션 하나의 필수 산출물

각 세션은 별도 데이터 파일과 별도 검증 기록을 갖는다.

1. 학습 목표 3~7개: 학습 후 독자가 설명하거나 구현할 수 있는 행동으로 작성한다.
2. 선수 지식: 용어 이름만 나열하지 않고 왜 필요한지 1~2문장으로 설명하고 해당 세션에 링크한다.
3. 정신 모델: 기술이 해결하는 문제, 내부에서 일어나는 일, 앞뒤 단계와의 관계를 그림 없이도 상상할 수 있게 설명한다.
4. 핵심 개념: 정의, 동작 순서, 규칙, 선택 기준, 반례를 포함한다.
5. 원본 기반 코드: 실제 `D:\dev` 학습자료의 파일 경로를 근거로 최소 2개 예제를 재구성한다.
6. 코드 해설: 중요한 줄과 표현을 왜 썼는지, 제거하거나 바꾸면 무엇이 달라지는지 설명한다.
7. 실행 방법: 필요한 버전, 명령, 입력, 실행 순서를 독립적으로 재현할 수 있게 적는다.
8. 실행 결과: 정상 결과를 최소 2개 제공하고 결과의 각 부분이 왜 나왔는지 해석한다.
9. 실패와 진단: 자주 만나는 오류·잘못된 결과·디버깅 순서를 최소 2개 제공한다.
10. 변형 실험: 값이나 구조를 바꾸었을 때 결과를 예측하고 확인하는 실험을 제공한다.
11. 실습: 따라 하기 1개, 응용 1개, 스스로 설계하기 1개 이상을 포함한다.
12. 전문가 관점: 성능, 보안, 테스트, 유지보수, 버전 차이 중 해당되는 내용을 다룬다.
13. 확인 문제와 체크포인트: 암기가 아니라 설명·예측·수정 능력을 확인한다.
14. 출처와 범위: 사용한 원본 파일, 공개 링크, 원본에서 다루지 않아 공식 문서로 보충한 범위를 구분한다.

## 2. 깊이 기준

- 한 세션은 하나의 핵심 질문에 답한다. 여러 큰 주제를 한 페이지에 묶지 않는다.
- 설명 본문은 원칙적으로 한국어 3,500자 이상을 목표로 하되 반복 문장으로 분량을 채우지 않는다.
- 섹션은 7개 이상, 코드 예제 2개 이상, 실행 결과 2개 이상, 진단 사례 2개 이상을 기본으로 한다.
- 코드만 보여 주지 않고 입력 → 처리 → 출력의 상태 변화를 추적한다.
- API 이름을 나열하는 대신 언제 선택하고 언제 피해야 하는지를 설명한다.
- 중급 이상 세션은 대안 비교와 trade-off를 포함한다.
- 고급·전문가 세션은 테스트 전략, 운영 실패, 성능 또는 보안 중 최소 하나를 실제 코드와 연결한다.

## 3. 세션별 독립 워크플로우

각 세션은 다음 순서로 처리한다.

1. **Source audit**: 관련 원본 파일을 전부 읽고 예제·주석·데이터·실행 흔적을 기록한다.
2. **Scope design**: 세션의 단일 핵심 질문과 선수·후속 세션을 확정한다.
3. **Draft**: 개념, 코드, 결과, 오류, 실습, 전문가 관점을 작성한다.
4. **Technical review**: 코드 문법, 버전, 결과 인과관계, 보안상 위험한 예제를 점검한다.
5. **Beginner review**: 이 세션만 처음 열어도 용어와 실행 순서를 따라갈 수 있는지 점검한다.
6. **Provenance review**: 원본 파일 경로와 공개 출처가 실제로 존재하며 비공개 링크나 비밀값이 없는지 확인한다.
7. **Schema validation**: 필수 필드, 최소 깊이, 링크, 중복 slug, 코드·결과 개수를 자동 검사한다.
8. **Build validation**: 정적 페이지 생성, 내부 링크, 검색 색인, 모바일 목차와 접근성을 검증한다.

## 4. 과정 전체의 완성 조건

- 인벤토리의 모든 의미 있는 원본 파일이 최소 한 세션에 연결된다.
- 같은 원본을 여러 세션에서 사용할 때는 서로 다른 학습 질문을 다룬다.
- `source coverage` 보고서에 미사용 파일과 이유가 남는다.
- 초급 → 기초 → 중급 → 고급 → 전문가 순서가 끊기지 않으며 모든 선수 링크가 실제 세션을 가리킨다.
- 과정마다 누적 프로젝트와 최종 프로젝트가 있고, 앞 세션의 지식을 다시 사용한다.
- 기존 25개 개요 페이지는 새 세션의 허브 또는 과정 소개로 전환하고 상세 설명을 대신하지 않는다.
